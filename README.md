# Электронный дневник школьника

Информационная система для ведения электронного дневника школьника с возможностью просмотра оценок, домашних заданий и расписания.

## Технологический стек

### Backend
- **Python 3.8+**
- **FastAPI** - современный веб-фреймворк для создания API
- **MongoDB** - NoSQL база данных
- **Motor** - асинхронный драйвер для MongoDB
- **PyJWT** - для аутентификации через JWT токены
- **Passlib** - для хеширования паролей

### Frontend
- **React** - библиотека для создания пользовательского интерфейса
- **React Router** - для маршрутизации
- **Axios** - для HTTP запросов к API

## Установка и запуск

### Backend

1. Перейдите в директорию backend:
```bash
cd backend
```

2. Создайте виртуальное окружение (рекомендуется):
```bash
python -m venv venv
```

3. Активируйте виртуальное окружение:
   - Windows:
   ```bash
   venv\Scripts\activate
   ```
   - Linux/Mac:
   ```bash
   source venv/bin/activate
   ```

4. Установите зависимости:
```bash
pip install -r requirements.txt
```

5. Настройте переменные окружения:
   - Скопируйте `.env.example` в `.env`
   - Убедитесь, что `MONGODB_URL` указывает на вашу MongoDB базу данных
   - При необходимости измените `SECRET_KEY` для безопасности
   - Укажите имя базы данных в `DATABASE_NAME` (по умолчанию: `diary_db`)

6. **Инициализация базы данных (опционально):**
   
   Для заполнения базы данных начальными тестовыми данными (предметы, классы, пользователи):
   ```bash
   python seed_data.py
   ```
   
   Это создаст:
   - Предметы (Математика, Русский язык, Английский язык и др.)
   - Классы (5А, 5Б, 10А)
   - Тестовых пользователей (администратор, учитель, ученик, родитель)
   - Примерные оценки, домашние задания и расписание
   
   **Важно:** Коллекции (таблицы) создаются автоматически при первом запуске приложения или при выполнении скрипта seed_data.py. Данные не заполняются автоматически - используйте seed_data.py для начальных данных.

7. Запустите сервер:
```bash
python run.py
```

Backend будет доступен по адресу: `http://localhost:8000`

API документация доступна по адресу: `http://localhost:8000/docs`

### Frontend

1. Перейдите в директорию frontend:
```bash
cd frontend
```

2. Установите зависимости:
```bash
npm install
```

3. Запустите приложение:
```bash
npm start
```

Frontend будет доступен по адресу: `http://localhost:3000`

## Функциональность

### Роли пользователей

1. **Ученик (Student)**
   - Просмотр своих оценок
   - Просмотр домашних заданий
   - Просмотр расписания

2. **Родитель (Parent)**
   - Просмотр оценок своих детей
   - Просмотр домашних заданий детей
   - Просмотр расписания детей

3. **Учитель (Teacher)**
   - Добавление оценок
   - Добавление домашних заданий
   - Просмотр информации о классах

4. **Администратор (Admin)**
   - Управление пользователями
   - Управление классами
   - Управление предметами
   - Полный доступ ко всем функциям

### Основные функции

- **Авторизация и регистрация** - система аутентификации через JWT токены
- **Просмотр оценок** - отображение оценок с возможностью фильтрации по предметам
- **Домашние задания** - просмотр заданий с указанием сроков сдачи
- **Расписание** - просмотр расписания уроков по дням недели
- **Дашборд** - главная страница с краткой информацией об оценках, заданиях и расписании

## API Endpoints

### Авторизация
- `POST /api/auth/register` - Регистрация нового пользователя
- `POST /api/auth/login` - Вход в систему
- `GET /api/auth/me` - Получение информации о текущем пользователе

### Пользователи
- `GET /api/users` - Список пользователей (только для админов и учителей)
- `GET /api/users/{id}` - Информация о пользователе

### Классы
- `GET /api/classes` - Список классов
- `GET /api/classes/{id}` - Информация о классе
- `POST /api/classes` - Создание класса (только для админов)

### Предметы
- `GET /api/subjects` - Список предметов
- `GET /api/subjects/{id}` - Информация о предмете
- `POST /api/subjects` - Создание предмета (только для админов)

### Оценки
- `GET /api/grades/student/{id}` - Оценки ученика
- `GET /api/grades/{id}` - Информация об оценке
- `POST /api/grades` - Создание оценки (только для учителей)

### Домашние задания
- `GET /api/homework/class/{id}` - Домашние задания класса
- `GET /api/homework/student/{id}` - Домашние задания ученика
- `GET /api/homework/{id}` - Информация о задании
- `POST /api/homework` - Создание задания (только для учителей)

### Расписание
- `GET /api/schedule/class/{id}` - Расписание класса
- `GET /api/schedule/student/{id}` - Расписание ученика
- `POST /api/schedule` - Создание элемента расписания (только для админов и учителей)

## База данных

Проект использует MongoDB. 

### Имя базы данных

По умолчанию используется имя `diary_db`. Вы можете изменить его в файле `.env`:
```
DATABASE_NAME=your_database_name
```

Рекомендуемые варианты названий:
- `diary_db` (текущее)
- `school_diary`
- `student_diary`
- `electronic_journal`

### Структура коллекций

Коллекции (аналог таблиц в SQL) создаются автоматически при первом обращении к ним. Структура коллекций:

- **users** - пользователи системы (ученики, учителя, родители, администраторы)
- **classes** - классы
- **subjects** - предметы
- **grades** - оценки
- **homework** - домашние задания
- **schedule** - расписание уроков

### Инициализация данных

**Коллекции создаются автоматически**, но **данные не заполняются автоматически**. 

Для заполнения начальными данными используйте скрипт:
```bash
python seed_data.py
```

Этот скрипт создаст:
- 8 предметов (Математика, Русский язык, Английский язык и др.)
- 3 класса (5А, 5Б, 10А)
- Тестовых пользователей всех ролей
- Примерные данные для демонстрации работы системы

После выполнения скрипта вы сможете войти в систему с тестовыми учетными данными (см. вывод скрипта).

## Безопасность

- Пароли хранятся в хешированном виде (bcrypt)
- Аутентификация через JWT токены
- Проверка прав доступа на уровне API
- CORS настроен для работы с фронтендом
